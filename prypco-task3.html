PRYPCOTrust & Compliance
5 / 12

Non-negotiable in a regulated market.

🛡 Trust

  • Disclosure on every output — "AI-generated guidance · Not financial advice" inline — not in a footer.
  • No commercial bias — Properties ranked by profile match, not inventory fill. Logic is auditable.
  • Data isolation — Profile and portfolio never used to train shared models. Tenant-of-one architecture.
  • Human escalation — Complex queries hand off to a licensed advisor with full AI context pre-loaded.

⚖️ Compliance

  • VARA & DLD guardrails — "Guaranteed" is programmatically blocked. All yields carry projection disclaimers.
  • RAG-grounded responses — V1: LLM constrained to PRYPCO + DLD verified data only. No hallucinated market claims.
  • Consent-gated AI — One-time acknowledgement before first interaction. Stored and auditable.
  • VARA Licence VL/25/05/001 — All AI guidance operates within licensed advisory scope.
PRYPCOResponse Flow

How a single response works.

Trust and compliance built in — not bolted on after.

1
User query
Types a question or taps a quick-reply chip
2
RAG retrieval
Fetches only verified PRYPCO + DLD data — no external web search
3
LLM generation
Claude responds within VARA-compliant system prompt with strict guardrails
4
Post-filter
Deterministic code blocks banned terms: guaranteed, price predictions, tax advice
5
Display + tags
Output shown with disclosure, 3-reason explainability tags, source citation
6
Audit log
Interaction logged: profile, query, model version, sources used, output text
The post-filter is a deterministic code-level check — not a prompt instruction. Language models can be coaxed. Code cannot.
