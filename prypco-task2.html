PRYPCOFramework

Five layers. Five different owners.

Each layer requires a different tool, owner, and validation approach.

1

Strings

AI (Claude API)
AST scanner extracts all JSX strings. Claude MT translates with fintech glossary. Human reviews.
2

Layout & RTL

Engineering

3

Typography

Design + Eng
Noto Sans Arabic loaded, line-height +20%, text-align flipped, bidi handling for mixed EN/AR.
4

Visual Assets

Design + AI
Images with embedded text replaced. Directional icons mirrored. Brand assets verified for Arabic variants.
5

Data & Formats

Engineering
Dates DD/MM/YYYY (Gregorian), currency 500 AED not AED 500, phone numbers LTR within RTL context.
PRYPCOVisual Assets
4 / 14

Visual & graphic asset localization.

Three categories — every image, icon and illustration must be assessed before launch.

Replace Images with embedded text

Property price banners, promo cards with 'AED 500', app store screenshots, onboarding illustrations with labels

Mirror Directional icons & UI elements

Back arrow (← becomes →), progress indicators (L→R becomes R→L), carousel pagination dots, timeline components

Audit Cultural & religious imagery

People imagery (conservative dress standards), financial lifestyle photos, holiday and seasonal visuals, trust badges and regulatory logos

Review with a native cultural consultant before launch — not a developer call.

PRYPCOAI Pipeline
5 / 14

The AI pipeline: 5 steps.

What would take weeks manually - done in 2 days with AI tooling.

01
AST Extract
jscodeshift
Scan all JSX/TSX files. Extract every hardcoded string with context: component name, file path, string type, and character limit. Output structured JSON.
→
02
MT via Claude
Claude API
Each string sent with full context: component, type, char limit, and fintech glossary. Claude translates with RTL markers. Returns confidence score per string.
→
03
Human Review
Native Speaker
2-hour session on D3. Reviews all strings for accuracy, tone, financial terminology.
→
04
Codemod
jscodeshift
Automated refactor replaces all hardcoded strings with i18n() calls. Zero manual edits.
→
05
RTL + Assets
Design + Eng
Layout mirrored. Icons flipped. Visual assets replaced. Typography switched to Noto Sans Arabic.

PRYPCOQuality Assurance
10 / 14

How we validate quality fast.

Automated first, human last — every layer is intentional.

1

AI visual review

Claude Vision checks every AR screen for overflow, clipping, English text, RTL issues.
2

AI back-translation check

Flags meaning drift across all strings before human review.
3

ESLint i18n rule

Any hardcoded string committed after sprint fails the build.
4

Native speaker string audit (D3)

Focused review on AI-flagged strings only.
5

Native speaker screen walkthrough (D9)

Signs off every screen on a real Arabic device.
6

Feature flag

Ships to Levant users only — EN users completely unaffected.
PRYPCODefinition of Done
11 / 14

Acceptance criteria.

A screen is not shipped until all pass.

Strings

  • Zero English fallbacks in AR locale
  • Financial terms match approved glossary
  • Plurals handled correctly
  • Error messages translated

Layout & RTL

  • All flex rows reversed
  • No LTR absolute positions
  • Icons and arrows mirrored
  • Navigation tab bar correct

Typography

  • Noto Sans Arabic loaded
  • No truncation on any string
  • Bidi text renders correctly
  • Line height sufficient for Arabic

Visual Assets

  • No images with embedded EN text
  • Directional icons mirrored
  • Culturally appropriate imagery
  • App Store AR screenshots updated

Performance

  • Arabic font no FOUT
  • <200ms launch parity
  • No AR-only crashes
  • Locale switch <500ms
